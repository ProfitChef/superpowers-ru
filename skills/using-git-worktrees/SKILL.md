# Использование Git Worktrees

## Обзор

Git worktree создают изолированные рабочие пространства, использующие один и тот же репозиторий, позволяя работать над несколькими ветками одновременно без переключения.

**Ключевой принцип:** Систематический выбор директории + проверка безопасности = надёжная изоляция.

**Объявите в начале:** «Я использую навык using-git-worktrees для настройки изолированного рабочего пространства.»

## Процесс выбора директории

Следуйте этому порядку приоритетов:

### 1. Проверьте существующие директории

```bash
# Проверить в порядке приоритета
ls -d .worktrees 2>/dev/null     # Предпочтительная (скрытая)
ls -d worktrees 2>/dev/null      # Альтернативная
```

**Если найдена:** Используйте эту директорию. Если обе существуют, `.worktrees` имеет приоритет.

### 2. Проверьте CLAUDE.md

```bash
grep -i "worktree.*director" CLAUDE.md 2>/dev/null
```

**Если предпочтение указано:** Используйте его, не спрашивая.

### 3. Спросите пользователя

Если директории нет и в CLAUDE.md нет предпочтений:

```
Директория для worktree не найдена. Где создавать worktree?

1. .worktrees/ (локально в проекте, скрытая)
2. ~/.config/superpowers/worktrees/<имя-проекта>/ (глобальное расположение)

Какой вариант предпочитаете?
```

## Проверка безопасности

### Для локальных директорий проекта (.worktrees или worktrees)

**ОБЯЗАТЕЛЬНО проверьте, что директория игнорируется, перед созданием worktree:**

```bash
# Проверить, игнорируется ли директория (учитывает local, global и system gitignore)
git check-ignore -q .worktrees 2>/dev/null || git check-ignore -q worktrees 2>/dev/null
```

**Если НЕ игнорируется:**

Согласно правилу «Чини сломанное немедленно»:
1. Добавьте соответствующую строку в .gitignore
2. Закоммитьте изменение
3. Продолжите создание worktree

**Почему это критично:** Предотвращает случайный commit содержимого worktree в репозиторий.

### Для глобальной директории (~/.config/superpowers/worktrees)

Проверка .gitignore не нужна — директория полностью вне проекта.

## Шаги создания

### 1. Определите имя проекта

```bash
project=$(basename "$(git rev-parse --show-toplevel)")
```

### 2. Создайте worktree

```bash
# Определить полный путь
case $LOCATION in
  .worktrees|worktrees)
    path="$LOCATION/$BRANCH_NAME"
    ;;
  ~/.config/superpowers/worktrees/*)
    path="~/.config/superpowers/worktrees/$project/$BRANCH_NAME"
    ;;
esac

# Создать worktree с новой веткой
git worktree add "$path" -b "$BRANCH_NAME"
cd "$path"
```

### 3. Запустите настройку проекта

Автоматически определите и запустите подходящую настройку:

```bash
# Node.js
if [ -f package.json ]; then npm install; fi

# Rust
if [ -f Cargo.toml ]; then cargo build; fi

# Python
if [ -f requirements.txt ]; then pip install -r requirements.txt; fi
if [ -f pyproject.toml ]; then poetry install; fi

# Go
if [ -f go.mod ]; then go mod download; fi
```

### 4. Проверьте чистую базовую линию

Запустите тесты, чтобы убедиться, что worktree стартует чистым:

```bash
# Примеры — используйте команду, подходящую для проекта
npm test
cargo test
pytest
go test ./...
```

**Если тесты падают:** Сообщите о сбоях, спросите — продолжать или расследовать.

**Если тесты проходят:** Сообщите о готовности.

### 5. Сообщите расположение

```
Worktree готов по адресу <полный-путь>
Тесты проходят (<N> тестов, 0 сбоев)
Готов к реализации <название-фичи>
```

## Краткий справочник

| Ситуация | Действие |
|----------|----------|
| `.worktrees/` существует | Использовать (проверить игнорирование) |
| `worktrees/` существует | Использовать (проверить игнорирование) |
| Обе существуют | Использовать `.worktrees/` |
| Ни одной нет | Проверить CLAUDE.md → Спросить пользователя |
| Директория не игнорируется | Добавить в .gitignore + commit |
| Тесты падают при проверке базовой линии | Сообщить о сбоях + спросить |
| Нет package.json/Cargo.toml | Пропустить установку зависимостей |

## Типичные ошибки

### Пропуск проверки игнорирования

- **Проблема:** Содержимое worktree отслеживается, загрязняет git status
- **Решение:** Всегда используйте `git check-ignore` перед созданием локального worktree

### Предположение о расположении директории

- **Проблема:** Создаёт несогласованность, нарушает соглашения проекта
- **Решение:** Следуйте приоритету: существующая > CLAUDE.md > спросить

### Продолжение с падающими тестами

- **Проблема:** Невозможно отличить новые баги от уже существующих
- **Решение:** Сообщите о сбоях, получите явное разрешение продолжить

### Жёсткое задание команд настройки

- **Проблема:** Ломается в проектах с другими инструментами
- **Решение:** Автоопределение по файлам проекта (package.json и т.д.)

```
[Создание worktree: git worktree add .worktrees/auth -b feature/auth]
[Запуск npm install]
[Запуск npm test — 47 проходят]

Worktree готов по адресу /Users/jesse/myproject/.worktrees/auth
Тесты проходят (47 тестов, 0 сбоев)
Готов к реализации фичи авторизации
```

## Красные флаги

**Никогда:**
- Создавать worktree без проверки, что он игнорируется (для локальных)
- Пропускать проверку базовой линии тестов
- Продолжать с падающими тестами без вопроса
- Предполагать расположение директории при неоднозначности
- Пропускать проверку CLAUDE.md

**Всегда:**
- Следуйте приоритету директорий: существующая > CLAUDE.md > спросить
- Проверяйте игнорирование для локальных директорий
- Автоопределяйте и запускайте настройку проекта
- Проверяйте чистую базовую линию тестов
