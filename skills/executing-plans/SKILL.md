# Исполнение планов

## Обзор

Загрузи план, критически оцени, выполняй задачи пакетами, отчитывайся для ревью между пакетами.

**Основной принцип:** Пакетное исполнение с контрольными точками для ревью архитектора.

**Объяви в начале:** «Я использую навык executing-plans для реализации этого плана.»

## Процесс

### Шаг 1: Загрузка и проверка плана
1. Прочитай файл плана
2. Критически оцени — определи вопросы или сомнения по плану
3. Если есть сомнения: Подними их перед партнёром-человеком до начала работы
4. Если сомнений нет: Создай TodoWrite и приступай

### Шаг 2: Выполни пакет
**По умолчанию: первые 3 задачи**

Для каждой задачи:
1. Отметь как in_progress
2. Следуй каждому шагу точно (план содержит пошаговые инструкции)
3. Запусти проверки, как указано
4. Отметь как completed

### Шаг 3: Отчёт
Когда пакет завершён:
- Покажи, что было реализовано
- Покажи результат проверок
- Скажи: «Готов к обратной связи.»

### Шаг 4: Продолжение
На основе обратной связи:
- Внеси изменения, если нужно
- Выполни следующий пакет
- Повторяй до завершения

### Шаг 5: Завершение разработки

После завершения и проверки всех задач:
- Объяви: «Я использую навык finishing-a-development-branch для завершения этой работы.»
- **ОБЯЗАТЕЛЬНЫЙ НАВЫК:** Используй superpowers:finishing-a-development-branch
- Следуй этому навыку для проверки тестов, представления вариантов, исполнения выбора

## Когда остановиться и попросить помощь

**НЕМЕДЛЕННО прекрати выполнение когда:**
- Столкнулся с блокером в середине пакета (отсутствует зависимость, тест падает, инструкция неясна)
- В плане критические пробелы, мешающие начать
- Ты не понимаешь инструкцию
- Проверка повторно не проходит

**Спроси уточнение, а не угадывай.**

## Когда возвращаться к предыдущим шагам

**Вернись к проверке (Шаг 1) когда:**
- Партнёр обновляет план на основе твоей обратной связи
- Фундаментальный подход требует пересмотра

**Не пробивайся через блокеры** — остановись и спроси.

## Запомни
- Сначала критически оцени план
- Следуй шагам плана точно
- Не пропускай проверки
- Ссылайся на навыки, когда план указывает
- Между пакетами: просто отчитайся и жди
- При блокере остановись, не угадывай
- Никогда не начинай реализацию на branch main/master без явного согласия пользователя

## Интеграция

**Необходимые навыки рабочего процесса:**
- **superpowers:using-git-worktrees** — ОБЯЗАТЕЛЬНО: Настрой изолированное рабочее пространство перед началом
- **superpowers:writing-plans** — Создаёт план, который этот навык исполняет
- **superpowers:finishing-a-development-branch** — Завершение разработки после всех задач
