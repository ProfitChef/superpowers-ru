# Завершение ветки разработки

## Обзор

Направляет завершение разработки, предлагая чёткие варианты действий и выполняя выбранный рабочий процесс.

**Ключевой принцип:** Проверить тесты → Предложить варианты → Выполнить выбор → Очистить.

**Объявите в начале:** «Я использую навык finishing-a-development-branch для завершения этой работы.»

## Процесс

### Шаг 1: Проверить тесты

**Перед предложением вариантов убедитесь, что тесты проходят:**

```bash
# Запустить тестовый набор проекта
npm test / cargo test / pytest / go test ./...
```

**Если тесты падают:**
```
Тесты не проходят (<N> сбоев). Необходимо исправить перед завершением:

[Показать сбои]

Невозможно продолжить с merge/PR пока тесты не пройдут.
```

Стоп. Не переходите к Шагу 2.

**Если тесты проходят:** Переходите к Шагу 2.

### Шаг 2: Определить базовую ветку

```bash
# Попробовать типичные базовые ветки
git merge-base HEAD main 2>/dev/null || git merge-base HEAD master 2>/dev/null
```

Или спросите: «Эта ветка отделилась от main — верно?»

### Шаг 3: Предложить варианты

Предложите ровно 4 варианта:

```
Реализация завершена. Что вы хотите сделать?

1. Выполнить merge в <базовую-ветку> локально
2. Запушить и создать Pull Request
3. Оставить ветку как есть (я разберусь позже)
4. Отбросить эту работу

Какой вариант?
```

**Не добавляйте пояснений** — варианты должны быть лаконичными.

### Шаг 4: Выполнить выбор

#### Вариант 1: Локальный merge

```bash
# Переключиться на базовую ветку
git checkout <базовая-ветка>

# Подтянуть последнюю версию
git pull

# Выполнить merge фича-ветки
git merge <фича-ветка>

# Проверить тесты на результате merge
<команда тестирования>

# Если тесты проходят
git branch -d <фича-ветка>
```

Далее: Очистка worktree (Шаг 5)

#### Вариант 2: Push и создание PR

```bash
# Запушить ветку
git push -u origin <фича-ветка>

# Создать PR
gh pr create --title "<заголовок>" --body "$(cat <<'EOF'
## Summary
<2-3 пункта об изменениях>

## Test Plan
- [ ] <шаги проверки>
EOF
)"
```

Далее: Очистка worktree (Шаг 5)

#### Вариант 3: Оставить как есть

Сообщите: «Ветка <имя> сохранена. Worktree оставлен по адресу <путь>.»

**Не очищайте worktree.**

#### Вариант 4: Отбросить

**Сначала подтвердите:**
```
Это безвозвратно удалит:
- Ветку <имя>
- Все коммиты: <список-коммитов>
- Worktree по адресу <путь>

Введите 'discard' для подтверждения.
```

Дождитесь точного подтверждения.

Если подтверждено:
```bash
git checkout <базовая-ветка>
git branch -D <фича-ветка>
```

Далее: Очистка worktree (Шаг 5)

### Шаг 5: Очистка worktree

**Для Вариантов 1, 2, 4:**

Проверьте, находитесь ли вы в worktree:
```bash
git worktree list | grep $(git branch --show-current)
```

Если да:
```bash
git worktree remove <путь-к-worktree>
```

**Для Варианта 3:** Оставьте worktree.

## Краткий справочник

| Вариант | Merge | Push | Сохранить Worktree | Очистить ветку |
|---------|-------|------|--------------------|----------------|
| 1. Локальный merge | ✓ | - | - | ✓ |
| 2. Создать PR | - | ✓ | ✓ | - |
| 3. Оставить как есть | - | - | ✓ | - |
| 4. Отбросить | - | - | - | ✓ (принудительно) |

## Типичные ошибки

**Пропуск проверки тестов**
- **Проблема:** Merge сломанного кода, создание падающего PR
- **Решение:** Всегда проверяйте тесты перед предложением вариантов

**Открытые вопросы**
- **Проблема:** «Что делать дальше?» → неоднозначно
- **Решение:** Предлагайте ровно 4 структурированных варианта

**Автоматическая очистка worktree**
- **Проблема:** Удаление worktree, когда он может понадобиться (Вариант 2, 3)
- **Решение:** Очищайте только для Вариантов 1 и 4

**Отсутствие подтверждения для отбрасывания**
- **Проблема:** Случайное удаление работы
- **Решение:** Требуйте набранное подтверждение «discard»

## Красные флаги

**Никогда:**
- Продолжать с падающими тестами
- Делать merge без проверки тестов на результате
- Удалять работу без подтверждения
- Делать force-push без явного запроса

**Всегда:**
- Проверяйте тесты перед предложением вариантов
- Предлагайте ровно 4 варианта
- Получайте набранное подтверждение для Варианта 4
- Очищайте worktree только для Вариантов 1 и 4
