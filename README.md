## Зачем нужна русская версия?

Вы общаетесь с Claude на русском. Всё идёт хорошо — пока не подгружаются скиллы.

**Проблема:** Оригинальные скиллы Superpowers написаны на английском. Когда агент загружает скилл, в контекст попадают тысячи токенов английского текста. После этого Claude неизбежно переключается на английский — даже если вы продолжаете писать по-русски.

**Решение:** Superpowers-RU — это полный перевод всех 14 скиллов и 21 вспомогательного файла на русский язык. Агент получает инструкции на русском, думает на русском, и отвечает на русском.

> Тот же фреймворк. Те же суперспособности. Только теперь — на вашем языке.

## Что это даёт?

### Мозговой штурм → Дизайн → План → Код

Агент не бросается писать код по первому слову. Сначала он уточняет задачу, предлагает 2-3 подхода с компромиссами, согласовывает дизайн по частям — и только потом составляет план реализации. Каждая задача в плане расписана настолько подробно, что её может выполнить джуниор без контекста проекта.

### Разработка через тесты (TDD)

RED → GREEN → REFACTOR — не рекомендация, а обязательный процесс. Тест пишется до кода. Код пишется ровно до прохождения теста. Потом refactoring. Потом commit.

### Систематическая отладка

Никаких «давай попробуем это» и случайных фиксов. Четыре фазы: расследование корневой причины → анализ паттернов → гипотеза и тестирование → исправление. Если вы не нашли корневую причину — вы не можете предлагать исправления.

### Субагенты и параллельная работа

Свежий субагент на каждую задачу — без загрязнения контекста. Двухэтапная проверка: сначала соответствие спецификации, потом качество кода. Несколько независимых проблем? Запускаем параллельных агентов.

### Часы автономной работы

После согласования плана Claude может работать автономно часами — выполняя задачу за задачей, проверяя каждую через ревью, не отклоняясь от плана.

## Установка

### Claude Code

```bash
git clone https://github.com/anthropics/superpowers-ru.git
cd superpowers-ru
./install.sh
```

Или вручную:

```bash
git clone https://github.com/anthropics/superpowers-ru.git ~/superpowers-ru
mkdir -p ~/.claude/skills
ln -sf ~/superpowers-ru/skills ~/.claude/skills/superpowers
```

### OpenCode

```bash
git clone https://github.com/anthropics/superpowers-ru.git ~/.config/opencode/superpowers
```

Затем подключите плагин и скиллы:

```bash
mkdir -p ~/.config/opencode/plugins ~/.config/opencode/skills
ln -sf ~/.config/opencode/superpowers/.opencode/plugins/superpowers.js ~/.config/opencode/plugins/superpowers.js
ln -sf ~/.config/opencode/superpowers/skills ~/.config/opencode/skills/superpowers
```

### Универсальный установщик

Скрипт `install.sh` автоматически определит установленные платформы:

```bash
git clone https://github.com/anthropics/superpowers-ru.git
cd superpowers-ru
chmod +x install.sh
./install.sh
```

Запустите новую сессию и спросите:

> «У тебя есть суперспособности?»

Агент должен подтвердить наличие скиллов и предложить их использовать.

## Библиотека скиллов

### Рабочий процесс

| Скилл | Описание |
|-------|----------|
| **brainstorming** | Превращает идею в проработанный дизайн через серию вопросов. Предлагает 2-3 подхода, согласовывает по частям, сохраняет дизайн-документ |
| **writing-plans** | Разбивает дизайн на задачи по 2-5 минут. Каждая задача — точные файлы, код, шаги проверки. Достаточно подробно для джуниора без контекста |
| **executing-plans** | Выполняет план пакетами по 3 задачи с контрольными точками для ревью |
| **subagent-driven-development** | Свежий субагент на каждую задачу + двухэтапная проверка (спецификация → качество кода) |
| **using-git-worktrees** | Создаёт изолированное рабочее пространство на отдельной ветке |
| **finishing-a-development-branch** | Проверяет тесты → предлагает 4 варианта (merge / PR / оставить / удалить) → выполняет выбор |

### Качество кода

| Скилл | Описание |
|-------|----------|
| **test-driven-development** | RED → GREEN → REFACTOR. Включает справочник антипаттернов тестирования |
| **systematic-debugging** | 4-фазный процесс поиска корневой причины. Включает техники трассировки, эшелонированной защиты, condition-based waiting |
| **verification-before-completion** | Запрещает заявлять «готово» без реального запуска проверок. Доказательства до утверждений |

### Совместная работа

| Скилл | Описание |
|-------|----------|
| **requesting-code-review** | Запрос ревью после каждой задачи. Шаблон для субагента-ревьюера |
| **receiving-code-review** | Получение фидбека: техническая проверка вместо слепого согласия. Pushback при необходимости |
| **dispatching-parallel-agents** | Параллельный запуск субагентов для независимых задач |

### Мета

| Скилл | Описание |
|-------|----------|
| **using-superpowers** | Bootstrap-скилл: как находить и использовать скиллы. Загружается автоматически |
| **writing-skills** | Создание новых скиллов по методологии TDD для документации |

## Как это работает

```
 1. brainstorming          Идея → уточнение → дизайн
          ↓
 2. using-git-worktrees    Изолированная ветка и рабочее пространство
          ↓
 3. writing-plans           Дизайн → подробный план задач
          ↓
 4. subagent-driven-dev    Субагент на задачу → двойное ревью → следующая задача
          ↓
 5. test-driven-dev        RED → GREEN → REFACTOR (внутри каждой задачи)
          ↓
 6. requesting-code-review  Ревью между задачами
          ↓
 7. finishing-branch        Тесты → merge / PR / оставить / удалить
```

**Скиллы активируются автоматически.** Агент проверяет релевантные скиллы перед каждым действием. Это обязательные процессы, а не рекомендации.

## Философия

| Принцип | Значение |
|---------|----------|
| **Тесты до кода** | Всегда пиши тест первым |
| **Система вместо хаоса** | Процесс важнее догадок |
| **Простота как цель** | Сложность — враг. YAGNI безжалостно |
| **Доказательства, не заявления** | Проверь перед тем как сказать «готово» |
